# 任务
根据随访对话，参考示例解析随访报告，最后给出结构化结果

# 要提取的字段
1. 随访信息来源
2. 受试者是否死亡
3. 受试者是否住院
4. 受试者是否手术
5. 受试者是否用药

# 字段定义及判定方法

## 1. 随访信息来源
- 定义：随访对象是否为本人
- 值域：["本人", "亲属"]
- 判定方法：综合对话内容，判断是否为本人接受随访

## 2. 受试者是否死亡
- 定义：受试者生命状态
- 值域：["是", "否"]
- 判定方法：
  - "是"：亲属明确提及受试者去世
  - "否"：其他所有情况

## 3. 受试者是否住院
- 定义：受试者是否接受住院治疗
- 值域：["是", "否", "未提及", "不确定"]
- 判定方法：
  - "是"：
    - 明确提及住院
    - 可推及住院的情况：支架手术、搭桥手术、瓣膜置换、脑血管造影DSA
  - "否"：
    - 明确否认住院
    - 提及未复查或仅健康体检
    - 仅提到造影但未明确是DSA还是CTA
    - 在社区医院就诊
  - "未提及"：无任何相关信息
  - "不确定"：受试者的逻辑混乱或自相矛盾，导致无法做出判断

## 4. 受试者是否手术
- 定义：受试者是否接受手术治疗
- 值域：["是", "否", "未提及", "不确定"]
- 判定方法：
  - "是"：
    - 明确提及手术
    - 明确的手术类型：支架、搭桥、瓣膜置换、脑血管造影DSA等
  - "否"：
    - 明确否认手术
    - 仅提及开药或保守治疗
  - "未提及"：无任何相关信息
  - "不确定"：受试者的逻辑混乱或自相矛盾，导致无法做出判断

## 5. 受试者是否用药
- 定义：受试者在随访期间是否使用药物
- 值域：["是", "否", "未提及", "不确定"]
- 判定方法：
  - "是"：随访期间使用药物（无论是否停药）
  - "否"：其他所有情况
  - "未提及"：无任何相关信息
  - "不确定"：受试者的逻辑混乱或自相矛盾，导致无法做出判断
 
# 输出结构
结构化结果，以JSON格式输出，确保其为一个可直接解析的字典（Dict[str, str]），格式如下：
```Json
{{
"随访信息来源":"XX",
"受试者是否死亡":"XX",
"受试者是否住院":"XX",
"受试者是否手术":"XX",
"受试者是否用药":"XX"
}}
```

# 注意事项
1. 四个事件（死亡、住院、手术、用药）应独立判断，不能互相推及。
2. 手术事件和住院事件虽独立判断，但某些特定手术可同时推及住院。
3. 特别地，如果'受试者是否死亡：'的预测结果显示为是，则只需保留以下两项信息：{{"随访信息来源":"XX"\n"受试者是否死亡":"XX"}}，不要加入多余的信息。
4. 请按照下面的示例，按照输出结构的要求输出结果。

# 示例
输入随访对话：
"角色1：喂你好。\n角色2：喂您好，请问您是？喂您好，请问您是######（音）吗？\n角色1：你好，哪里？\n角色2：我这边是南京第一医院的，您去年不是6月5号的时候在我们这边做了一个冠脉的检查，我对您近期的情况进行一下随访。\n角色1：哦。\n角色2：您最近情况怎么样？\n角色1：我说后来就没有检查过了。\n角色2：没有是吧？您最近感觉怎么样？会不会有胸痛，胸闷不舒服？\n角色1：我一直在吃药。\n角色2：吃药以后会不会有胸痛，胸闷不舒服呢？\n角色1：没有。\n角色2：都挺好的是吧？\n角色1：对。\n角色2：从去年到现在没有复查过是吧？\n角色1：对。\n角色2：有没有体检过？\n角色1：没有。\n角色2：也没有复查也没有体检，然后吃了药感觉还挺好的？\n角色1：是，对。\n角色2：也没有做过什么手术吧？\n角色1：没有。\n角色2：您现在吃哪些药？\n角色1：我就吃他那个叫什么？\n角色2：他汀。\n角色1：嗯。\n角色2：然后就吃一个他汀吗？\n角色1：对，就是（00:01:20）的。\n角色2：就吃一个他汀，是吧？\n角色1：对。\n角色2：然后最近情况也蛮好的，也没有什么不舒服？\n角色1：是的。\n角色2：好的，好，谢谢您的配合。\n00:01:36\n完"
输出结果：
```json
{{
"随访信息来源": "本人"
"受试者是否死亡": "否"
"受试者是否住院": "否"
```
